## :desc
[Docker](https://ru.wikipedia.org/wiki/Docker) - это открытая платформа для создания, развертывания и управления контейнерами. Контейнеры позволяют упаковывать приложения со всеми их зависимостями в стандартизированные единицы для обеспечения простоты развертывания и переноса между различными средами.

# Какая команда используется для создания нового образа из Dockerfile?
- [ ] `docker run`
- [ ] `docker create`
- [x] `docker build`
- [ ] `docker commit`

## :desc
Команда `docker build` используется для создания Docker-образа из `Dockerfile` и контекста. `Dockerfile` содержит набор инструкций, которые Docker использует для создания образа. Контекст обычно представляет собой текущий каталог и все его подкаталоги.

Пример использования: `docker build -t <image_name> .`

Наиболее полезные параметры команды `docker build`:
  - `-t, --tag` - Позволяет задать имя и тег для создаваемого образа.
  - `-f, --file` - Указывает путь к Dockerfile, если он находится не в корне контекста сборки.
  - `--build-arg` - Позволяет передавать аргументы в Dockerfile во время сборки.

---

# Какой командой можно создать новый контейнер из образа в Docker?
- [ ] `docker build`
- [ ] `docker create`
- [x] `docker run`
- [ ] `docker start`

## :desc
Команда `docker run` создает и запускает новый контейнер из образа. Она используется для выполнения команд в новом контейнере.

Пример использования: `docker run -d --name <container_name> <image_name>`

Наиболее полезные параметры команды `docker run`:
  - `-d, --detach` - Запускает контейнер в фоновом режиме (detach).
  - `--name` - Присваивает имя контейнеру.
  - `-p, --publish` - Публикует порты контейнера на хосте.

# Какая команда используется для отображения списка всех контейнеров?
- [x] `docker ps`
- [ ] `docker images`
- [ ] `docker list`
- [ ] `docker containers`

## :desc
Команда `docker ps` выводит список запущенных контейнеров. С ключом `-a` можно увидеть все контейнеры, включая остановленные.

Пример использования: `docker ps -a`

Наиболее полезные параметры команды `docker ps`:
  - `-a, --all` - Показать все контейнеры (включая остановленные).
  - `-q, --quiet` - Выводить только идентификаторы контейнеров.
  - `--filter` - Фильтровать контейнеры по заданным критериям.

# Какая команда используется для остановки работающего контейнера?
- [ ] `docker kill`
- [ ] `docker pause`
- [x] `docker stop`
- [ ] `docker remove`

## :desc
Команда `docker stop` останавливает работающий контейнер, корректно завершая его процессы.

Пример использования: `docker stop <container_name>`

Наиболее полезные параметры команды `docker stop`:

# Какой командой можно удалить ненужный образ в Docker?
- [ ] `docker rmi`
- [ ] `docker rm`
- [x] `docker image rm`
- [ ] `docker image remove`

## :desc
Команда `docker image rm` используется для удаления образов. Она позволяет освободить место и управлять репозиторием образов.

Пример использования: `docker image rm <image_name>`

Наиболее полезные параметры команды `docker image rm`:
  - `-f, --force` - Принудительно удалить образ.
  - `--no-prune` - Не удалять неиспользуемые родительские образы.

---

# Какая команда используется для входа в работающий контейнер?
- [x] `docker exec -it <container_id> bash`
- [ ] `docker attach <container_id>`
- [ ] `docker run -it <container_id> bash`
- [ ] `docker login <container_id>`

## :desc
Команда `docker exec -it <container_id> bash` позволяет войти в работающий контейнер и использовать указанную командную оболочку. В данном случае `bash`.

Наиболее полезные параметры команды `docker exec`:
  - `-i, --interactive` - Запуск в интерактивном режиме.
  - `-t, --tty` - Включение псевдотерминала.

# Какая команда используется для просмотра логов контейнера?
- [ ] `docker logs`
- [ ] `docker events <container_id>`
- [x] `docker logs <container_id>`
- [ ] `docker history`

## :desc
Команда `docker logs <container_id>` выводит логи указанного контейнера, что позволяет отслеживать его поведение и диагностику.

Наиболее полезные параметры команды `docker logs`:
  - `-f, --follow` - Следить за обновлениями логов в реальном времени.
  - `--tail` - Выводить только последние строки логов.

# Какой командой можно сохранить контейнер в виде образа?
- [x] `docker commit`
- [ ] `docker save`
- [ ] `docker export`
- [ ] `docker image save`

## :desc
Команда `docker commit` создает новый образ из контейнера, фиксируя его текущее состояние.

Пример использования: `docker commit <container_id> <image_name>`

Наиболее полезные параметры команды `docker commit`:
  - `-a, --author` - Указать автора коммита.
  - `-m, --message` - Добавить сообщение к коммиту.
